# PR 拆分计划

目标仓库：https://github.com/20240520zhaolongye/qiniuyun

原则：

- 每个 PR 只实现或修改一个单一功能。
- PR 尽可能小，合并后主分支必须保持可运行。
- 每个 PR 合并前至少运行 `npm test`；涉及 C++ 后端的 PR 还要运行 `npm run build`。
- 不提交 `build/`、日志文件、IDE 配置和临时文件。

## PR 01：初始化项目结构与基础页面

标题：初始化 SpriteForge 前端页面和项目脚本

功能描述：建立项目基础结构，提供 `index.html`、`src/styles.css`、`src/app.js`、`src/renderer.js`、`src/core.js` 和 `package.json`，让用户可以打开基础页面并看到素材生成工具界面。

实现思路：使用原生 HTML/CSS/JavaScript 实现浏览器界面，不引入前端框架，降低依赖复杂度。

测试方式：运行 `npm start`，打开 `http://127.0.0.1:5173`，确认页面可访问。

## PR 02：实现前端素材参数表单

标题：新增素材生成参数表单

功能描述：新增素材名称、文本描述、素材类型、风格、尺寸、视角、动画、帧数、FPS 和导出目标等输入控件。

实现思路：通过 DOM 查询读取表单值，组装生成请求对象，为后续生成接口和渲染逻辑提供统一输入。

测试方式：打开页面，修改各控件，确认不会报错且表单值可参与生成。

## PR 03：实现 Canvas 单帧素材渲染

标题：新增 Canvas 单帧素材预览

功能描述：根据生成计划在 Canvas 上绘制怪物、角色、道具、图标、地块和特效的单帧预览。

实现思路：在 `src/renderer.js` 中封装 `renderFrame`，使用 Canvas 2D API 绘制不同素材类型。

测试方式：运行项目，切换素材类型，确认预览区域能显示对应图形。

## PR 04：实现 Sprite Sheet 排列

标题：新增 Sprite Sheet 自动排版预览

功能描述：根据帧数自动横向排列 Sprite Sheet，便于游戏引擎导入。

实现思路：在 `src/renderer.js` 中封装 `renderSheet`，循环绘制每一帧并按帧宽偏移。

测试方式：修改帧数为 1、4、8，确认 Sprite Sheet 宽度和帧排列正确。

## PR 05：实现动画播放与透明背景切换

标题：新增动画播放和透明背景预览

功能描述：支持点击“播放”循环预览帧动画，并支持切换透明棋盘背景和纯色背景。

实现思路：使用 `setInterval` 按 FPS 切换当前帧，通过 CSS class 切换预览区背景。

测试方式：设置帧数大于 1，点击“播放”确认动画连续；点击“透明”确认背景切换。

## PR 06：实现前端导出下载

标题：新增 PNG、Sprite Sheet 和 JSON 下载

功能描述：支持下载当前帧 PNG、完整 Sprite Sheet PNG 和元数据 JSON。

实现思路：使用离屏 Canvas 生成图片数据，通过 Blob 和临时下载链接触发文件下载。

测试方式：分别点击三个下载按钮，确认文件能下载且内容可打开。

## PR 07：新增 C++ 核心生成模块

标题：新增 C++ 素材计划生成核心

功能描述：用 C++ 实现尺寸解析、文件名清洗、色板规范化、Prompt 构建、随机种子和导出 JSON 生成。

实现思路：在 `src_cpp/core.cpp` 和 `src_cpp/core.h` 中集中处理生成逻辑，前端后续通过 API 调用。

测试方式：运行 `npm run build`，确认 C++ 代码可编译。

## PR 08：新增轻量 JSON 解析与序列化

标题：新增 C++ JSON 解析和序列化工具

功能描述：为 C++ 后端提供请求解析和响应构造能力，不依赖第三方 JSON 库。

实现思路：在 `src_cpp/json.h` 中实现基本 JSON value、parser 和 stringify，覆盖对象、数组、字符串、数字、布尔值和 null。

测试方式：运行 `npm run build` 和 `npm test`，确认 API 请求能被解析。

## PR 09：实现 C++ 本地 HTTP 服务

标题：新增 C++ 本地 HTTP 静态服务

功能描述：用 C++ 提供本地 HTTP 服务，支持访问首页和前端静态资源。

实现思路：基于 Windows socket 实现监听、请求读取、响应写入、Content-Type 和路径限制。

测试方式：运行 `npm start`，打开 `http://127.0.0.1:5173`，确认页面可访问。

## PR 10：新增 `/api/plan` 生成接口

标题：新增 C++ `/api/plan` 素材计划接口

功能描述：前端提交参数后，C++ 后端返回 seed、Prompt、metadata、draw 配置和导出 JSON。

实现思路：服务端解析 POST JSON，请求进入 `create_asset_plan`，再返回浏览器渲染所需结构。

测试方式：运行 `npm test`，确认 `/api/plan` 返回 200 且包含 `metadata.assetName` 和 Prompt。

## PR 11：前端切换为调用 C++ 生成接口

标题：将前端生成逻辑切换到 C++ API

功能描述：前端不再本地生成素材计划，而是调用 `/api/plan` 获取生成结果。

实现思路：在 `src/core.js` 中封装 `createAssetPlan` 的 fetch 调用，在 `src/app.js` 中处理异步生成状态。

测试方式：运行项目，点击“生成素材”，确认状态显示 Generated by C++ 且预览正常。

## PR 12：新增 `/api/health` 健康检查接口

标题：新增 C++ 服务健康检查接口

功能描述：提供 `/api/health`，用于确认 C++ 服务正常运行。

实现思路：在 HTTP 路由中处理 GET `/api/health`，返回服务状态、名称和版本。

测试方式：访问 `http://127.0.0.1:5173/api/health`，确认返回 `status: ok`。

## PR 13：新增构建和启动脚本

标题：新增 C++ 构建与启动脚本

功能描述：提供 `npm run build` 和 `npm start`，让评委能一键构建并启动项目。

实现思路：通过 `scripts/build-cpp.ps1` 调用 g++ 构建，通过 `scripts/start-cpp.ps1` 构建后运行 exe。

测试方式：运行 `npm run build` 和 `npm start`，确认构建产物生成且服务启动。

## PR 14：新增自动化测试

标题：新增前端工具函数和 C++ 服务测试

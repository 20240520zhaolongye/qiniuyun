# 题目二：2D 游戏素材生成工具项目开发文档

## 项目名称

**SpriteForge：面向 2D 游戏开发的智能素材生成工具**

## 一、项目背景

2D 游戏开发中，美术素材通常包括角色、敌人、道具、地形块、UI 图标、技能特效、场景背景、动画帧、表情、头像等。独立开发者和小团队经常面临以下问题：

- 美术成本高，外包周期长。
- 素材风格不统一，难以直接用于项目。
- AI 生成图片虽然方便，但往往不符合游戏工程需求，例如尺寸不规范、背景不透明、动画不连续、无法导出 Sprite Sheet。
- 主流 2D 引擎如 Unity、Godot、Cocos Creator、GameMaker 等需要规范化资源格式。

因此，本项目开发一个工具，让用户通过文本描述或简单参数，快速生成可直接导入游戏引擎的 2D 游戏素材。

## 二、项目目标

开发一个低成本、高效率、风格一致、工程友好的 2D 游戏素材生成工具。

核心目标：

1. 用户输入文本或选择参数，即可生成 2D 游戏素材。
2. 支持角色、道具、地图块、UI 图标、特效、背景等常见素材类型。
3. 支持统一风格控制，保证同一项目内素材视觉一致。
4. 支持导出 PNG、透明背景图、Sprite Sheet、Tilemap、JSON 元数据等。
5. 可无缝融入 Unity、Godot、Cocos Creator 等主流 2D 游戏开发流程。
6. 尽量降低生成成本，通过模板化 Prompt、批量生成、缓存、局部重绘等方式提升效率。

## 三、目标用户

- 独立游戏开发者
- 游戏 Jam 参赛者
- 小型游戏工作室
- 游戏策划与原型设计人员
- 教学、课程设计、毕设项目团队

## 四、核心功能

### 1. 文本生成素材

用户输入自然语言，例如：

> 生成一个像素风的蓝色史莱姆怪物，适合横版动作游戏，32x32 像素，透明背景。

系统自动生成对应素材，并提供多张候选图。

### 2. 参数化生成

用户也可以通过简单参数生成素材：

- 素材类型：角色 / 怪物 / 道具 / 地图块 / UI / 技能特效 / 背景
- 游戏类型：RPG / 平台跳跃 / 塔防 / Roguelike / 解谜 / 休闲
- 美术风格：像素风 / 手绘风 / 卡通风 / 暗黑风 / Q版 / 低饱和幻想风
- 尺寸：16x16、32x32、64x64、128x128、512x512
- 视角：正面 / 侧面 / 俯视 / 等距视角
- 背景：透明 / 纯色 / 场景背景
- 动画：静态 / 待机 / 行走 / 攻击 / 死亡 / 技能释放

### 3. 风格一致性控制

建立“项目风格档案”，包括：

- 主色板
- 线条粗细
- 明暗层级
- 像素密度
- 角色比例
- 世界观关键词
- 参考图或种子图
- 固定负面 Prompt

生成新素材时自动套用该风格档案，保证同一游戏中的素材不会明显割裂。

### 4. 动画帧生成

支持生成常见动画：

- 角色待机 Idle
- 行走 Walk
- 跑步 Run
- 攻击 Attack
- 受击 Hit
- 死亡 Death
- 技能 Cast
- 特效爆炸 Explosion

导出格式：

- 单帧 PNG
- 横向 Sprite Sheet
- 网格 Sprite Sheet
- Aseprite 可编辑格式，作为扩展功能
- JSON 动画元数据

### 5. Tilemap 与地图块生成

支持生成：

- 草地、泥土、水面、石路、墙体、悬崖等地形块
- 自动拼接瓦片 Auto Tile
- 16x16 / 32x32 / 64x64 Tile Set
- 支持 Godot TileSet、Unity Tile Palette 使用

### 6. UI 与图标生成

支持生成：

- 技能图标
- 背包图标
- 装备图标
- 按钮
- 血条、蓝条、经验条
- 游戏 Logo 草图
- 菜单面板背景

### 7. 素材后处理

生成后自动处理：

- 去背景 / 透明化
- 裁剪空白边缘
- 缩放到标准尺寸
- 像素风最近邻缩放
- 自动居中
- 色板统一
- 边缘清理
- Sprite Sheet 排版
- 文件命名规范化

### 8. 引擎导出适配

支持导出：

```text
/export
  /unity
    player_idle.png
    player_idle.json
    tileset.png
    tileset.meta.json
  /godot
    player_idle.png
    player_idle.tres.json
  /cocos
    player_idle.png
    player_idle.plist.json
```

导出信息包括：

- 帧宽
- 帧高
- 帧数
- 帧间隔
- Pivot 点
- Pixels Per Unit
- 碰撞框建议
- 动画名称

## 五、用户流程

1. 用户创建项目，选择游戏类型和美术风格。
2. 系统生成项目风格档案。
3. 用户选择素材类型，例如“怪物”。
4. 用户输入文本描述或设置参数。
5. 系统生成 Prompt，并调用图像生成模型。
6. 用户查看候选结果，选择满意版本。
7. 系统执行透明化、裁剪、尺寸规范化、Sprite Sheet 生成。
8. 用户导出到 Unity / Godot / Cocos 对应目录。
9. 后续生成素材继续沿用项目风格档案。

## 六、系统架构

```text

```

## 七、推荐技术栈

本项目建议采用“前端 Web 工具 + 后端生成服务 + 图像后处理模块 + 游戏引擎导出适配”的技术架构。MVP 阶段优先选择开发成本低、生态成熟、便于展示的技术组合。

### 前端

- React + TypeScript：构建交互式 Web 界面，提升代码可维护性。
- Vite：作为前端构建工具，启动快，适合原型开发。
- Tailwind CSS：快速搭建整洁统一的界面样式。
- Canvas / Konva.js：用于素材预览、裁剪、缩放、Sprite Sheet 预览和简单编辑。
- Zustand：管理生成参数、项目风格档案、生成历史和导出状态。

### 后端

- Python FastAPI：提供素材生成、Prompt 构建、风格档案管理、导出等 API。
- Pillow：用于图片裁剪、缩放、透明背景处理、自动居中和 Sprite Sheet 合成。
- OpenCV：用于更复杂的图像处理，例如边缘清理、背景分割和轮廓检测，可作为扩展能力。
- SQLite：MVP 阶段用于存储项目、素材记录、风格档案和生成历史。
- PostgreSQL：后续多人协作或云端版本可替换为 PostgreSQL。
- Celery + Redis：后续用于异步生成任务、批量生成和任务队列。
- 本地文件存储：MVP 阶段直接保存 PNG、Sprite Sheet、JSON 元数据。
- S3 兼容对象存储：后续云端部署时用于素材文件管理。

### AI 生成

可支持多种模型接口：

- Mock AI Generator：MVP 阶段可先用占位图或预设样例模拟生成流程，保证产品逻辑完整。
- Stable Diffusion 系列
- SDXL
- Flux 系列
- ComfyUI API
- 本地 LoRA 风格模型
- 第三方图像生成 API

### 游戏引擎适配

- Unity：PNG + JSON + Sprite Sheet
- Godot：PNG + JSON / Resource 配置
- Cocos Creator：PNG + plist / JSON

### 推荐 MVP 技术组合

```text
前端：React + TypeScript + Vite + Tailwind CSS
状态管理：Zustand
预览编辑：Canvas / Konva.js
后端：Python FastAPI
图像处理：Pillow
数据库：SQLite
AI 生成：Mock AI Generator，后续接入 ComfyUI API / Stable Diffusion / Flux
导出格式：PNG + Sprite Sheet + JSON 元数据
```

选择该组合的原因：

1. 开发速度快，适合快速完成可演示原型。
2. 前后端职责清晰，便于后续接入真实 AI 图像生成服务。
3. Pillow 能覆盖 MVP 阶段大部分图像后处理需求。
4. SQLite 部署简单，适合单机版本和课程项目。
5. 导出 PNG、Sprite Sheet、JSON 元数据即可满足 Unity、Godot、Cocos Creator 的基础接入需求。

## 九、Prompt 生成策略

系统不直接把用户输入丢给模型，而是转换为结构化 Prompt。

### Prompt 组成

```text
素材类型 + 角色/物体描述 + 游戏视角 + 美术风格 + 尺寸约束 + 背景要求 + 动画要求 + 风格一致性描述 + 负面约束
```

### 示例

```text
A 32x32 pixel art blue slime monster for a 2D side-scrolling fantasy RPG game, cute round body, simple smiling face, clean dark outline, limited color palette, consistent lighting, centered sprite, transparent background, game-ready asset, idle animation frame.
Negative prompt: blurry, realistic, 3D, complex background, text, watermark, inconsistent outline, extra limbs.
```

## 十、成本控制方案

1. 使用项目风格模板减少重复调参。
2. 对常见素材类型预设 Prompt 模板。
3. 支持批量生成，减少单次人工操作成本。
4. 缓存相同参数的生成结果。
5. 优先生成低分辨率草图，确认后再高清细化。
6. 对局部问题使用局部重绘，而不是整图重生成。
7. 对像素素材使用小尺寸生成加后处理，不盲目生成大图。
8. 对团队项目可训练轻量 LoRA 保持风格统一。

## 十一、质量评价指标

| 指标 | 说明 |
|---|---|
| 可用性 | 是否能直接导入游戏引擎 |
| 风格一致性 | 是否符合项目整体美术风格 |
| 清晰度 | 边缘、轮廓、主体是否清楚 |
| 尺寸规范 | 是否符合 16x16、32x32、64x64 等标准 |
| 透明背景 | 是否满足游戏素材透明需求 |
| 动画连续性 | 帧与帧之间是否自然 |
| 生成效率 | 单个素材生成和处理耗时 |
| 成本 | 单张素材平均生成成本 |

# 项目开发 Prompt

下面这个 Prompt 可以直接交给 AI 编程助手或开发团队，用于生成项目原型。

```text
请开发一个名为 SpriteForge 的 2D 游戏素材生成工具原型。

项目目标：
用户可以通过文本描述或简单参数，快速生成适合 2D 游戏开发的素材，并导出为主流游戏引擎可用的格式。工具需要重点考虑生成效率、素材质量、风格一致性和游戏开发流程适配。

核心功能：
1. 提供一个 Web 界面，用户可以创建素材生成任务。
2. 用户可输入自然语言描述，也可选择参数：
   - 素材类型：角色、怪物、道具、UI 图标、Tile、背景
   - 美术风格：像素风、卡通风、手绘风、暗黑风、Q版
   - 尺寸：16x16、32x32、64x64、128x128、512x512
   - 视角：正面、侧面、俯视、等距视角
   - 背景：透明、纯色、场景背景
   - 动画类型：静态、待机、行走、攻击、死亡

4. 支持项目风格档案，包含色板、线条风格、光照方式、视角、负面 Prompt 等信息。
5. 调用图像生成接口生成素材。如果没有真实 API，可以先用 Mock 数据和占位图实现流程。
6. 对生成结果进行后处理：
   - 透明背景
   - 自动裁剪
   - 尺寸规范化
   - 居中
   - Sprite Sheet 排版
7. 支持导出：
   - PNG
   - Sprite Sheet
   - JSON 元数据
8. JSON 元数据需包含：
   - assetName
   - assetType
   - frameWidth
   - frameHeight
   - frameCount
   - animationName
   - fps
   - pivot
   - exportTarget
9. 界面需要包含：
   - 左侧参数面板
   - 中间素材预览区
   - 右侧项目风格档案区
   - 底部导出按钮和生成记录
10. 需要提供良好的代码结构，便于后续接入 Stable Diffusion、Flux 或其他图像生成 API。

技术要求：
- 前端可使用 React + TypeScript。
- 后端可使用 Python FastAPI 或 Node.js Express。
- 图像处理可预留 Pillow / OpenCV 接口。

请输出：
1. 完整项目目录结构。
2. 前端主要页面和组件代码。
3. 后端 API 代码。
4. Prompt 构建逻辑。
5. 示例风格档案。
6. 示例导出 JSON。
7. README 使用说明。
```

# 素材生成 Prompt 模板

如果要直接生成游戏素材，可以使用下面这个模板。

```text
Generate a {size} {style} 2D game asset.

Asset type: {asset_type}
Description: {description}
Game genre: {game_genre}
Camera view: {view}
Background: transparent
Art direction: {art_direction}
Color palette: {color_palette}
Line style: {line_style}
Lighting: {lighting}
Animation: {animation_type}, frame {frame_index} of {frame_count}

Requirements:
- game-ready 2D asset
- centered composition
- clean readable silhouette
- consistent style
- transparent background
- suitable for Unity, Godot, and Cocos Creator
- no text, no watermark, no complex background

Negative prompt:
blurry, realistic photo, 3D render, inconsistent style, noisy details, bad anatomy, extra limbs, cropped object, shadow background, watermark, text, logo
```
